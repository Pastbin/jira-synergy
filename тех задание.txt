

Техническое задание на дипломный проект
"Разработка веб-приложения для управления проектами и задачами"

1. Введение
1.1. Наименование проекта: "Jira Synergy"
1.2. Назначение и цель проекта: Разработка полнофункционального веб-приложения для управления проектами и задачами по методологии Kanban с элементами scrum. Цель — демонстрация навыков fullstack-разработки на современном стеке технологий (Next.js, TypeScript, Node.js, PostgreSQL, Socket.io, Docker).

2. Функциональные требования
2.1. Модуль аутентификации и авторизации:

Регистрация и вход по email/паролю.

Разделение прав доступа: Пользователь имеет полный доступ только к своим проектам.

2.2. Модуль управления проектами:

Создание, просмотр, редактирование и удаление проектов.

Возможность приглашения других пользователей в проект (по email или ссылке).

2.3. Модуль управления задачами (Kanban-доска):

Создание, просмотр, редактирование и удаление задач внутри проекта.

Стандартные статусы задач: To Do (К выполнению)	🔄 In Progress (В работе)	👀 Review (На ревью)	🧪 Testing (На тестировании)	✅ Done (Готово)

Возможность перетаскивания задач между колонками статусов (drag-and-drop, реализовать с помощью @hello-pangea/dnd или аналогичной библиотеки).

Назначение ответственного исполнителя на задачу (из числа участников проекта).

У задачи должны быть: название, описание, статус, исполнитель, дата создания, дедлайн (опционально).

2.4. Модуль уведомлений:

Всплывающие (toast) уведомления внутри приложения о важных событиях (например, "Вас назначили на задачу", "Задача переведена в статус 'Готово'").

Реализация с помощью Context API или RTK Query.

2.5. Модуль аналитики (опционально, но желательно для солидности):

Простая диаграмма Ганта, отображающая задачи на временной шкале (можно использовать библиотеку frappe-gantt или аналогичную).

Статистика по проекту: количество задач по статусам.

2.6. Режим реального времени (WebSockets):

Использование Socket.io для синхронизации изменений на Kanban-доске между всеми участниками проекта в реальном времени без перезагрузки страницы.

3. Нефункциональные требования

Производительность: Время первой загрузки страницы не более 3 секунд.

Безопасность: Валидация всех входящих данных на сервере, хеширование паролей, защита от XSS и SQL-инъекций.

Масштабируемость: Архитектура приложения должна позволять добавлять новые модули.

Кроссплатформенность: Адаптивный интерфейс для корректного отображения на desktop и mobile devices.

Юзабилити: Интуитивно понятный и современный пользовательский интерфейс.

4. Технологический стек

Frontend: Next.js 15+, React 18+, TypeScript, Redux Toolkit, Material-UI, Socket.io-client.

Backend: Next.js API Routes, TypeORM.

База данных: PostgreSQL.

Аутентификация: NextAuth.js.

Реальное время: Socket.io.

Деплой: VPS, Nginx, Docker (опционально), GitHub Actions.

Тестирование: Jest, React test library и другие

5. Этапы разработки

Проектирование архитектуры БД.

Настройка проекта и базовой конфигурации.

Разработка core-функционала (аутентификация, CRUD проектов).

Разработка модуля задач (Kanban).

Интеграция Socket.io для real-time updates.

Разработка модуля уведомлений и аналитики.

Тестирование, исправление ошибок.

Деплой на VPS, настройка CI/CD.

Написание пояснительной записки.

6. Требования к оформлению

Использование TypeScript строго обязательно.

Код должен быть покрыт комментариями.

6. Тестирование:

Проект должен быть покрыт максимально тестами
Работа с базами данных, unit тестирование компонентов, ui, возможно e2e тестирование.
